# UX & CRO Guidelines

## Supersymmetry Shopify Theme

---

# Purpose

The purpose of this document is to define the user experience and conversion principles for every customer-facing interface in this repository.

Every UI decision should reduce friction.

Every interaction should increase confidence.

Every page should move the customer one step closer to completing a purchase.

This document takes precedence whenever multiple UX implementations are possible.

---

# Core Philosophy

Customers should never have to think.

The interface should answer questions before they are asked.

Every click should feel expected.

Every interaction should provide feedback.

The customer should always know:

• where they are

• what happened

• what happens next

Confusion is conversion loss.

---

# The Conversion Funnel

Optimize in this order.

```
Discover

↓

Understand

↓

Trust

↓

Desire

↓

Add to Cart

↓

Checkout

↓

Purchase

↓

Return
```

Never optimize a later stage at the expense of an earlier one.

Example:

Do not aggressively upsell if it distracts from completing checkout.

---

# Primary KPI

Primary KPI:

Completed Purchases

Secondary KPIs:

Checkout Started

Add to Cart Rate

Product Engagement

Average Order Value

Return Customer Rate

Do not increase friction to improve secondary KPIs.

# Product Page Principles

The product page exists for one purpose:

Increase purchase confidence.

Every section should answer a customer question.

If a section doesn't reduce uncertainty or increase desire, reconsider its existence.

---

# CTA Hierarchy

There should only ever be one primary CTA.

Everything else is secondary.

The customer should never wonder:

"What am I supposed to click?"

Avoid competing actions.

Avoid multiple accent colors.

Avoid multiple dominant buttons.

---

# Add to Cart

The Add to Cart button should always be obvious.

Requirements:

Large

Easy to reach

Visible

Fast

Responsive

Never hide purchasing behind unnecessary interactions.

---

# Sticky Add to Cart

Sticky ATC exists for convenience.

Not persuasion.

It should appear only after the primary purchase area is no longer visible.

It should disappear when unnecessary.

Never obstruct content.

---

# Variant Selection

Variant selection should reduce anxiety.

Unavailable options should remain visible but disabled.

Customers should understand why something cannot be selected.

Avoid confusing state changes.

---

# Inventory Messaging

Use inventory messaging only when truthful.

Avoid fake urgency.

Examples:

✓ Only 2 left

✓ Low stock

Avoid:

Selling fast!

Almost gone!

Unless backed by real inventory.

---

# Pricing

Price should never compete with the product.

Discounts should remain understandable.

Avoid multiple simultaneous promotions.

Customers should immediately understand:

Current price

Original price

Savings

Nothing else.

---

# Shipping

Shipping information should appear before checkout.

Customers should never need to search for delivery information.

Estimated delivery should feel specific.

Good:

Delivered Jul 14–16

Poor:

Ships soon

---

# Returns

Returns should reduce purchase anxiety.

Keep messaging concise.

Avoid legal language.

Customers want reassurance.

Not policy documents.

---

# Cart Philosophy

The cart is not storage.

The cart is the final confidence-building step before checkout.

The customer should immediately understand:

What they added

What it costs

When it arrives

How to continue

Nothing else should distract from these questions.

---

# Cart Drawer

The drawer should:

Appear instantly

Confirm success

Show updated totals

Remain responsive

Allow quantity changes

Allow removal

Keep Checkout visible

Avoid unnecessary content.

---

# Checkout CTA

Checkout should remain the strongest action.

There should never be another equally prominent button.

Closing the drawer is sufficient.

Avoid "Continue Shopping" as a competing primary CTA.

---

# Quantity Controls

Quantity changes should feel instantaneous.

Never reload the page.

Always provide visual feedback.

Handle errors gracefully.

---

# Loading States

Every asynchronous action requires feedback.

Examples:

Spinner

Progress indicator

Button state

Skeleton

Never leave the customer wondering if something happened.

---

# Empty States

Every empty state should explain:

What happened

Why

What to do next

Avoid dead ends.

---

# Error Messages

Errors should be:

Clear

Specific

Actionable

Helpful

Avoid:

Unknown error

Something went wrong

Instead:

"This size is no longer available."

---

# Forms

Every form field increases friction.

Request only necessary information.

Reduce typing.

Use appropriate input types.

Support autofill.

---

# Mobile First

Design for thumbs.

Primary actions belong in comfortable reach.

Avoid tiny controls.

Avoid horizontal scrolling.

Avoid hover-only interactions.

---

# Scrolling

Scrolling is acceptable.

Searching is not.

Customers should naturally encounter important information.

Never hide essential information in deep accordions.

---

# Animations

Animations should communicate.

Not decorate.

Good uses:

Drawer opening

Confirmation

Loading

State transitions

Avoid decorative animations.

---

# Speed

Perceived speed matters more than actual speed.

Show feedback immediately.

Render progressively.

Never freeze the interface.

---

# Visual Hierarchy

Use size.

Whitespace.

Contrast.

Typography.

Avoid relying on color alone.

---

# Trust

Trust comes from design quality.

Not badges.

Avoid clutter.

Avoid excessive icons.

Premium brands rely on clarity.

---

# Typography

Readable.

Consistent.

Generous spacing.

Never sacrifice readability for style.

---

# Accessibility

Good accessibility improves conversions.

Support:

Keyboard

Screen readers

Reduced motion

Visible focus

Large touch targets

Semantic HTML

---

# Merchant Experience

Merchants should understand every setting.

Avoid confusing terminology.

Use sensible defaults.

Avoid unnecessary configuration.

---

# Design Philosophy

Supersymmetry is not a discount marketplace.

It is a premium fashion brand.

The interface should feel:

Elegant

Quiet

Confident

Editorial

Modern

Intentional

Avoid visual noise.

Avoid promotional overload.

Whitespace is a feature.

---

# Decision Framework

Before implementing any UI ask:

Does this reduce uncertainty?

Does this increase confidence?

Does this remove friction?

Does this make purchasing easier?

Does this improve mobile?

Would removing this improve the experience?

If removing it improves the experience,

remove it.

---

# Final Principle

The highest-converting interfaces rarely contain the most elements.

They contain only the right ones.

When in doubt,

simplify.
